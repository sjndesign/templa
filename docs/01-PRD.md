# PRD (Product Requirements Document)

## 1. 프로젝트 개요
- Templa는 디자인 시스템을 기반으로 다양한 브랜드 템플릿을 자동 생성하고, 모든 결과물이 동일한 디자인 규칙을 유지하는 디자인 자동화 서비스이다.
- 사용자는 디자인 경험 없이도 브랜드를 빠르고 일관되게 제작할 수 있다.

---

## 2. 프로젝트 목표
- 디자인 시스템만 입력하면 일관된 브랜드 결과물을 생성한다.
- 비디자이너도 전문적인 브랜드 결과물을 만들 수 있도록 한다.
- 다양한 포맷(PNG, PDF, PPTX 등)으로 다운로드를 지원한다.
- 반복적인 디자인 작업을 제거하여 제작 속도를 극대화한다.

---

## 3. 문제 정의
- 디자이너가 매번 새로운 디자인을 반복 제작한다.
- 같은 브랜드라도 결과물이 매번 다르다.
- SNS, 문서, 웹 디자인이 분리되어 관리된다.
- 수정 작업마다 전체 디자인을 다시 만들어야 한다.

---

## 4. 타겟 사용자
- 1인 사업자
- 스타트업 초기 팀
- 마케팅 담당자
- 디자이너가 없는 중소기업
- 개발자 (브랜드 디자인 직접 해야 하는 사람)

---

## 5. 핵심 기능
- 디자인 시스템 생성 (컬러 / 폰트 / 스타일 정의)
- 브랜드 정보 입력 (회사명 / 슬로건 / 로고)
- 템플릿 자동 생성
  - 명함
  - 이메일 서명
  - SNS 프로필 이미지
- 실시간 미리보기
- 다운로드 (PNG, PDF)

---

## 6. MVP 범위
- 디자인 시스템 입력 폼
- 브랜드 정보 입력
- 템플릿 생성 기능
  - 명함
  - 이메일 서명
  - SNS 프로필 이미지
- PNG 다운로드

---

## 7. 사용자 흐름
1. 사용자가 디자인 시스템 입력
2. 브랜드 정보 입력
3. 생성할 템플릿 선택
4. "생성하기" 버튼 클릭
5. 시스템이 템플릿 자동 생성
6. 미리보기 확인
7. PNG / PDF 다운로드

---

## 8. 입력 데이터 구조

사용자가 입력하는 데이터는 다음과 같다:

- brandName (브랜드명)
- tagline (슬로건)
- logo (이미지)
- primaryColor
- secondaryColor
- fontFamily
- brandTone (minimal / bold / playful / corporate)

이 데이터는 모든 템플릿 생성의 기준이 된다.

---

## 9. 템플릿 생성 방식

- 디자인 시스템 데이터를 기반으로 템플릿을 생성한다.
- 모든 템플릿은 동일한 디자인 규칙을 따른다.
- 컴포넌트 기반 구조로 렌더링된다.

구성 요소:
- Button Component
- Card Component
- Layout System

렌더링 방식:
- HTML + CSS 기반 렌더링
- Canvas 기반 이미지 생성 (향후)
- PDF export 지원

---

## 10. 향후 확장 계획

### V1 - 브랜딩 템플릿
- 명함 템플릿 생성
- 이메일 서명 템플릿 생성
- SNS 프로필 생성 (블로그, 인스타그램, 유튜브)

### V2 - SNS & 마케팅 템플릿
- 블로그 대표 이미지 및 카드뉴스 생성
- 인스타그램 피드/스토리 템플릿 생성
- 유튜브 썸네일 및 채널아트 생성
- 브랜드 스타일 자동 적용 유지

### V3 - 마케팅/비즈니스 확장
- 전단지, 배너, 광고 이미지 생성
- 랜딩페이지 디자인 자동 생성
- 간단한 홈페이지 템플릿 생성 (HTML 기반)

### V4 - 회사 문서 자동화
- 제안서, PPT 템플릿 자동 생성
- 견적서, 발주서 등 비즈니스 문서 템플릿
- PDF 및 PPTX 다운로드 지원

### V5 - 웹사이트 생성 기능
- 디자인 시스템 기반 랜딩페이지 자동 생성
- HTML/CSS 또는 React 코드로 export
- 반응형 웹 자동 적용

### V6 - 디자인 시스템 플랫폼 (핵심 비전)
- 하나의 디자인 시스템으로 모든 콘텐츠 자동 생성
- 브랜드 스타일 중앙 관리
- 팀 단위 협업 기능
- 템플릿 마켓플레이스 확장

# F-007 브랜드 삭제

## 1. 기능 개요

사용자가 생성한 브랜드를 휴지통으로 이동하여 삭제할 수 있도록 한다.

---

## 2. 목적

- 사용자가 더 이상 사용하지 않는 브랜드를 삭제할 수 있도록 한다.
- 실수로 삭제한 브랜드를 복원할 수 있도록 휴지통으로 이동한다.

---

## 3. 관련 화면

| 화면 ID | 화면명 |
|----------|---------|
| SC-006 | 브랜드 목록 |
| SC-008 | 브랜드 상세 |
| SC-015 | 휴지통 |

---

## 4. 사용자

- 로그인한 사용자

---

## 5. 입력 데이터

| 항목 | 타입 | 필수 | 설명 |
|------|------|------|------|
| brandIds | Number[] | O | 삭제할 브랜드 ID 목록 |

---

## 6. 처리 과정

1. 브랜드 목록 또는 브랜드 상세 화면에서 삭제할 브랜드를 선택한다.
2. 삭제하기 버튼을 클릭한다.
3. 삭제 확인 팝업을 표시한다.
4. 사용자가 삭제를 확인한다.
5. 시스템이 브랜드를 휴지통으로 이동한다.
6. 시스템이 연결된 디자인 시스템을 함께 휴지통으로 이동한다.
7. 시스템이 연결된 템플릿을 함께 휴지통으로 이동한다.
8. 삭제 완료 메시지를 표시한다.
9. 브랜드 목록을 새로 조회한다.

---

## 7. 출력 결과

### 성공

- 브랜드가 휴지통으로 이동되었습니다.
- 30일 이내에 복원할 수 있습니다.
- 브랜드 목록이 갱신됩니다.

### 실패

- 브랜드 삭제에 실패했습니다.

---

## 8. 예외 처리

| 상황 | 처리 |
|------|------|
| 삭제할 브랜드가 없음 | 삭제할 브랜드를 선택해주세요. |
| 이미 삭제된 브랜드 | 이미 삭제된 브랜드입니다. |
| 브랜드에 접근 권한이 없음 | 삭제 권한이 없습니다. |
| 서버 오류 | 잠시 후 다시 시도해주세요. |

---

## 14. 비즈니스 규칙

- 브랜드 삭제 시 연결된 디자인 시스템도 함께 휴지통으로 이동한다.
- 브랜드 삭제 시 연결된 모든 템플릿도 함께 휴지통으로 이동한다.
- 휴지통에서 브랜드를 복원하면 디자인 시스템과 템플릿도 함께 복원된다.
- 휴지통에서 브랜드를 영구 삭제하면 연결된 모든 데이터도 함께 영구 삭제된다.
- 휴지통에 보관된 데이터는 30일 후 자동 삭제된다.

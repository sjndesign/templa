# F-002 회원가입

## 1. 기능 개요

사용자가 새로운 계정을 생성하여 Templa 서비스를 이용할 수 있도록 한다.

---

## 2. 목적

- 새로운 사용자 계정을 생성한다.
- 회원 정보를 저장한다.
- 회원가입 완료 후 로그인할 수 있도록 한다.

---

## 3. 관련 화면

| 화면 ID | 화면명 |
|----------|---------|
| SC-003 | 회원가입 |
| SC-002 | 로그인 |

---

## 4. 사용자

- 비회원

---

## 5. 입력 데이터

| 항목 | 타입 | 필수 | 설명 |
|------|------|------|------|
| 이름 | String | O | 가입할 회원의 성함 |
| 이메일 | String | O | 계정 아이디 |
| 비밀번호 | String | O | 계정 비밀번호 |
| 비밀번호 확인 | String | O | 비밀번호 확인 |

---

## 6. 처리 과정

1. 이름을 입력한다.
2. 이메일을 입력한다.
3. 이메일 중복 확인을 수행한다.
4. 비밀번호를 입력한다.
5. 비밀번호 확인을 입력한다.
6. 가입하기 버튼을 클릭한다.
7. 시스템이 입력값을 검증한다.
8. 시스템이 회원 정보를 저장한다.
9. 회원가입 완료 메시지를 표시한다.
10. 로그인 화면으로 이동한다.

---

## 7. 출력 결과

### 성공

- 회원가입 완료 메시지
- 로그인 화면으로 이동

### 실패

- 오류 메시지를 표시한다.
- 회원가입 화면을 유지한다.

---

## 8. 예외 처리

| 상황 | 처리 |
|------|------|
| 이름 미입력 | 이름을 입력해주세요. |
| 아이디 미입력 | 아이디를 입력해주세요. |
| 아이디 중복 | 이미 사용 중인 아이디입니다. |
| 이메일 미입력 | 이메일을 입력해주세요. |
| 이메일 형식 오류 | 이메일 형식을 확인해주세요. |
| 비밀번호 미입력 | 비밀번호를 입력해주세요. |
| 비밀번호 불일치 | 비밀번호가 일치하지 않습니다. |
| 비밀번호 길이 부족 | 8자 이상 입력해주세요. |
| 서버 오류 | 잠시 후 다시 시도해주세요. |

---

## 9. 권한

- 비회원만 접근 가능
- 로그인된 사용자는 대시보드로 이동한다.

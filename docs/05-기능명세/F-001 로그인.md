# F-001 로그인

## 1. 기능 개요

사용자가 이메일과 비밀번호를 입력하여 서비스에 로그인한다.

---

## 2. 목적

- 인증된 사용자만 서비스를 이용할 수 있도록 한다.
- 로그인 성공 시 대시보드로 이동한다.

---

## 3. 관련 화면

| 화면 ID | 화면명 |
|----------|---------|
| SC-002 | 로그인 |
| SC-005 | 대시보드 |

---

## 4. 사용자

- 비회원

---

## 5. 입력 데이터

| 항목 | 타입 | 필수 | 설명 |
|------|------|------|------|
| 이메일 | String | O | 가입한 이메일 |
| 비밀번호 | String | O | 계정 비밀번호 |

---

## 6. 처리 과정

1. 사용자가 이메일을 입력한다.
2. 사용자가 비밀번호를 입력한다.
3. 로그인 버튼을 클릭한다.
4. 입력값을 검증한다.
5. 서버에 로그인 요청을 보낸다.
6. 인증 성공 여부를 확인한다.
7. 인증에 성공하면 Access Token을 저장한다.
8. 대시보드로 이동한다.

---

## 7. 출력 결과

### 성공

- 로그인 성공 메시지
- Access Token 저장
- 대시보드(SC-005) 이동

### 실패

- 이메일 또는 비밀번호가 올바르지 않습니다.
- 로그인에 실패했습니다.

---

## 8. 예외 처리

| 상황 | 처리 |
|------|------|
| 이메일 미입력 | "이메일을 입력해주세요." |
| 비밀번호 미입력 | "비밀번호를 입력해주세요." |
| 이메일 형식 오류 | 이메일 형식을 확인해주세요. |
| 존재하지 않는 계정 | 가입된 계정이 없습니다. |
| 비밀번호 불일치 | 비밀번호가 올바르지 않습니다. |
| 서버 오류 | 잠시 후 다시 시도해주세요. |

---

## 9. 권한

- 비회원만 접근 가능
- 로그인된 사용자는 로그인 페이지 접근 시 대시보드로 이동한다.
